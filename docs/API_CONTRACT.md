# waliFit — API Contract

Canonical inventory of every backend HTTP endpoint, the Zod schema that validates its body, the response shape, and the mobile hook that calls it. **This is the artifact route-scaffold work consumes** — when adding a route, update this file; when wiring a hook, find your route here.

> Generated by reading `backend/src/routes/*.ts`, `packages/shared/src/schemas/*.ts`, `react-native/hooks/*.ts`, and `react-native/lib/api.ts`. If this drifts from those sources, the source files win — fix the doc.

## Conventions

- **Base URL:** `EXPO_PUBLIC_API_URL` on mobile, `config.PORT` on backend (default `:4000`). No `/api/v1/` prefix today — routes are mounted at the root by `server.ts`.
- **Auth:** every route except `/healthz` is protected by `requireAuth` (`backend/src/lib/auth.ts`) which verifies a Supabase JWT and sets `request.user = { id, email }`.
- **All requests + responses:** JSON. Mobile sends `Authorization: Bearer <jwt>` via `lib/api.ts`.
- **Error shape:** `{ error: 'invalid_body' | 'invalid_query' | 'invalid_date' | 'forbidden' | 'workout_not_found' | ..., issues?: ZodFlatError }` with appropriate 4xx code.
- **IDs:** PostgreSQL UUIDs (`@db.Uuid`) to match Supabase `auth.users`.
- **Units:** all weights in kg, all durations in seconds. UI converts on display.
- **Dates:** path/query params use `YYYY-MM-DD`. Response timestamps are ISO 8601.

## Mobile call layer

Two functions in `react-native/lib/api.ts`:

- `apiQuery<T>(path, init?)` — reads. Throws `ApiError` on 4xx/5xx, `NetworkError` on offline.
- `apiMutate<T>({ method, path, body })` — writes. Tries to send; on network failure or 5xx, hands off to `lib/syncQueue.ts` for retry. Returns `{ kind: 'sent', data } | { kind: 'queued', queuedId }`.

Hooks in `react-native/hooks/useMutations.ts` wrap `apiMutate` for each write and invalidate the `home/snapshot` query on success.

## Shared schemas

Zod schemas live in `packages/shared/src/schemas/` (`workout.ts`, `nutrition.ts`, `vitality.ts`). Both packages consume them via the workspace dep `"walifit-shared": "file:../packages/shared"`:

```ts
// backend (validates incoming bodies)
import { StartWorkoutSchema } from 'walifit-shared';
const parsed = StartWorkoutSchema.safeParse(request.body);

// mobile (types only — no runtime validation, no Zod execution on the device)
import type { StartWorkoutBody } from 'walifit-shared';
```

Mobile resolves the package because `react-native/metro.config.js` adds the repo root to `watchFolders` (Metro otherwise refuses to bundle files outside the project). Hierarchical `node_modules` lookup is left at its default — overriding it breaks transitive resolution (e.g. `react-native-reanimated` → `semver`).

**Zod version note.** `react-native/package.json` is on `zod ^4` while `backend/` and `walifit-shared` peer-dep are on `zod ^3`. Mobile uses `import type` only, so the device never executes the shared schemas at runtime — the version mismatch is currently invisible. If mobile ever needs to call `.safeParse()` against a shared schema, align the versions first.

|---|---|---|---|---|---|

## When adding a new route — checklist

1. Add or extend a Zod schema in `packages/shared/src/schemas/`.
2. Add the handler in `backend/src/routes/<file>.ts`. Use `requireAuth` preHandler for anything user-scoped.
3. Validate the body with `Schema.safeParse(request.body)` and return `{ error: 'invalid_body', issues }` with 400 on failure.
4. Register the route file in `backend/src/server.ts` if it's a new file.
5. Add a row to the relevant table above in this doc.
6. Mirror the body type in `react-native/hooks/useMutations.ts` (until Metro resolves `walifit-shared`).
7. Add a hook in `react-native/hooks/` that calls `apiQuery` (read) or `apiMutate` (write). Mutations must go through `apiMutate` — never raw `fetch`.
8. If the change should reflect on Home immediately, invalidate `['home', 'snapshot']` in `onSuccess`.
